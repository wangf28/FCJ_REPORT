Events Participated

The information below documents all events I participated in during my AWS FCJ internship from September to December 2025. Each event provided valuable learning experiences and opportunities to connect with the AWS community.

Event 1: FCJ 2025 Kickoff Event

Event Name: First Cloud Journey 2025 - Kickoff Event

Date & Time: Full Day, September 8, 2025

Location: Ho Chi Minh City, Vietnam

Role: Participating Intern

Event Overview

The FCJ 2025 Kickoff Event marked the official beginning of the First Cloud Journey internship program. This event brought together all selected interns, program coordinators, mentors, and AWS community leaders to launch the 2025 cohort. It was an intensive orientation and networking session designed to help interns understand program expectations, connect with peers, and present initial project ideas.

Key Topics Covered

  • FCJ Program Structure: Timeline, milestones, evaluation criteria, and expectations
  • AWS Learning Path: Recommended certifications and learning resources for interns
  • Project Guidelines: Requirements for internship projects, documentation standards, and deliverables
  • Mentorship Framework: How to work with mentors and coordinators throughout the program
  • Community Engagement: AWS Vietnam community activities and networking opportunities

Sessions Attended

  1. Welcome and Program Overview - FCJ Program Director

    • Introduction to FCJ history and mission
    • 2025 program structure and timeline
    • Success stories from previous cohorts
    • Evaluation criteria and graduation requirements

  2. Project Ideation Workshop - Technical Mentors

    • Brainstorming project topics aligned with AWS services
    • Identifying real-world problems to solve
    • Scoping projects appropriately for 4-month timeline
    • Technical feasibility assessment

  3. Intern Project Presentations - All Participating Interns

    • 5-minute pitch of initial project ideas
    • Q&A with mentors and fellow interns
    • Feedback on project scope and technical approach
    • Networking and finding potential collaborators

  4. AWS Community Introduction - AWS User Group Leaders

    • Overview of AWS Vietnam community activities
    • Upcoming events and workshops
    • How to contribute to the community
    • Building professional networks in cloud computing

Outcomes and Value Gained

  • Program Clarity: Gained complete understanding of FCJ program structure, expectations, and success criteria
  • Project Validation: Presented IoT Security Infrastructure project idea and received positive feedback from mentors
  • Peer Connections: Met 25+ fellow interns from various universities, established collaboration channels
  • Mentor Network: Connected with 5 technical mentors specializing in AWS security and IoT
  • Motivation Boost: Inspired by success stories from previous FCJ cohorts who went on to AWS careers

Impact on Internship Journey

This kickoff event set the foundation for my entire internship:

  • Validated project topic choice and received guidance on scoping
  • Established relationships with mentors who provided ongoing support
  • Understood program milestones and planned accordingly
  • Gained confidence from seeing the supportive FCJ community

Event 5: AWS Cloud Mastery Series #2 - Infrastructure as Code

Event Name: AWS Cloud Mastery Series #2 - Infrastructure as Code Best Practices

Date & Time: 10:00 AM, November 20, 2025

Location: Virtual Event (AWS Webinar)

Role: Attendee

Event Overview

The second installment of the AWS Cloud Mastery Series focused on Infrastructure as Code (IaC) best practices, multi-region deployment strategies, and cost optimization techniques. This webinar was particularly timely as I was implementing Terraform modules for my project during the same week.

Key Topics Covered

  • Infrastructure as Code Fundamentals: Comparing CloudFormation, Terraform, and CDK approaches
  • Terraform Best Practices: Module design, state management, and workspace strategies
  • Multi-Region Deployments: Strategies for global application distribution and disaster recovery
  • Cost Optimization: Right-sizing resources, using spot instances, and implementing cost allocation tags
  • CI/CD for Infrastructure: Automating infrastructure deployments with testing and validation

Sessions Attended

  1. Terraform Module Design Patterns - AWS DevOps Specialist

    • Creating reusable, composable Terraform modules
    • Managing module versioning and dependencies
    • Remote state management with S3 and DynamoDB locking
    • Workspace strategies for multi-environment deployments

  2. Multi-Region Architecture - AWS Solutions Architect

    • Active-active vs active-passive multi-region patterns
    • Data replication strategies with DynamoDB Global Tables
    • Route53 routing policies for global traffic management
    • Cross-region disaster recovery planning

  3. Cost Optimization Strategies - AWS Cost Optimization Specialist

    • Identifying cost optimization opportunities
    • Implementing AWS Cost Explorer and Budgets
    • Tagging strategies for cost allocation
    • Reserved Instances and Savings Plans

Outcomes and Value Gained

  • IaC Expertise: Validated my Terraform implementation approach and learned advanced module patterns
  • Scalability Planning: Understood how to design for multi-region expansion from the start
  • Cost Awareness: Implemented cost monitoring and optimization practices in my project
  • DevOps Skills: Learned to build robust CI/CD pipelines for infrastructure deployment
  • Best Practices: Applied AWS-recommended patterns for state management and module organization

Practical Applications in Internship

  • Terraform Modules: Refactored project infrastructure into reusable modules (ACM Private CA, Lambda, DynamoDB, IoT Core)
  • Remote State: Configured S3 backend with DynamoDB locking for team collaboration
  • Cost Tags: Implemented comprehensive tagging strategy for resource tracking and cost allocation
  • CI/CD Pipeline: Built GitHub Actions workflow with Terraform plan/apply automation
  • Testing: Added terraform validate and tflint checks to pre-deployment pipeline

Impact on Project Quality

  • Infrastructure Modularity: 60% reduction in infrastructure code duplication through module reuse
  • Deployment Reliability: Zero-downtime deployments with proper state locking and planning
  • Cost Visibility: Real-time cost tracking by environment and service component
  • Team Collaboration: Multiple team members can safely work on infrastructure simultaneously
